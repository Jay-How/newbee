# 回合制卡牌游戏开发文档

## 1. 项目概述

### 1.1 游戏简介
本项目是一款原创回合制卡牌游戏，基于Godot引擎开发。游戏采用规则书中定义的"彼岸史诗：战团"战斗规则，玩家作为远征队指挥官，构筑卡组、指挥战团护送圣物，借助圣物能量击败所有敌人与最终BOSS。

### 1.2 核心玩法特点
- 原创区域部署机制，强调单位位置与移动的策略性
- 圣物能量作为核心资源，影响法强值计算
- 前后排部署策略，提升战术深度
- 多样化的卡牌类型（单位卡、策略卡）与技能效果
- 随机生成式肉鸽地图系统，提升游戏可玩性

## 4. 技术实现路径

### 4.1 开发流程
1. **项目初始化**：创建Godot项目，设置项目配置
2. **核心系统开发**：实现卡牌系统、战斗系统、玩家系统和AI系统
3. **场景开发**：创建战斗场景、地图场景和菜单场景
4. **卡牌数据设计**：设计并实现卡牌数据结构
5. **AI逻辑实现**：实现AI决策和行为逻辑
6. **UI开发**：开发游戏界面和交互系统
7. **测试与调优**：测试游戏功能，优化性能

### 4.2 关键技术点
- **卡牌系统**：使用Godot的资源系统存储卡牌数据，实现卡牌的创建、使用和管理
- **战斗系统**：使用状态机管理战斗流程，处理回合逻辑和战斗计算
- **UI系统**：使用Godot的CanvasLayer和Control节点构建游戏界面
- **数据管理**：使用Godot的ConfigFile和Resource系统管理游戏数据
- **AI系统**：使用决策树和评估函数实现AI行为
- **词条系统**：实现词条的触发逻辑和管理，支持技能的多样化效果
- **效果系统**：实现效果的执行和管理，处理技能触发后的具体效果

### 4.3 性能优化
- **对象池**：使用对象池减少卡牌和单位的创建和销毁
- **批处理**：使用批处理减少绘制调用
- **资源预加载**：预加载常用资源，减少运行时加载时间
- **LOD**：对3D模型使用LOD技术，优化渲染性能

### 5.2 回合流程控制
```gdscript
# src/core/BattleSystem.gd
func start_turn() -> void:
    # 重置行动点
    action_points = max_action_points
    
    # 计算法强值
    magic_power = calculate_magic_power()
    
    # 触发回合开始效果
    trigger_turn_start_effects()
    
    # 设置战斗状态
    battle_state = "player_turn"

func end_turn() -> void:
    # 触发回合结束效果
    trigger_turn_end_effects()
    
    # 切换到敌方回合
    battle_state = "enemy_turn"
    process_enemy_turn()

func process_enemy_turn() -> void:
    # 敌方行动逻辑
    var ai_system = AISystem.new()
    var decision = ai_system.make_decision(enemy_data, {
        "player_data": player_data,
        "relic_data": relic_data,
        "magic_power": magic_power
    })
    
    # 执行敌方决策
    execute_enemy_decision(decision)
    
    # 检查战斗结果
    if check_victory():
        battle_state = "victory"
    elif check_defeat():
        battle_state = "defeat"
    else:
        # 切换回玩家回合
        start_turn()
```

### 5.3 战斗逻辑处理
```gdscript
# src/core/BattleSystem.gd
func handle_attack(attacker: Node, target: Node) -> void:
    # 计算伤害
    var damage = calculate_damage(attacker, target)
    
    # 应用伤害
    apply_damage(target, damage)
    
    # 检查目标状态
    if target.health <= 0:
        handle_unit_death(target)
    elif target.health == 1:
        handle_unit_injured(target)

func calculate_damage(attacker: Node, target: Node) -> int:
    # 计算伤害逻辑
    var damage = attacker.attack_power
    
    # 考虑防御和抗性
    damage = max(1, damage - target.defense)
    
    return damage

func apply_damage(target: Node, damage: int) -> void:
    # 应用伤害逻辑
    target.health -= damage

func handle_unit_death(unit: Node) -> void:
    # 处理单位死亡逻辑
    unit.queue_free()
    
    # 触发死亡效果
    trigger_death_effects(unit)

func handle_unit_injured(unit: Node) -> void:
    # 处理单位受伤逻辑
    unit.is_injured = true
    unit.queue_free()
    
    # 加入弃牌堆
    if unit.is_player_unit:
        player_data.discard_pile.append(unit.card_id)
    else:
        enemy_data.discard_pile.append(unit.card_id)
```
